[13] IMPORT EXPORT

Lorsque vous avez terminé, exportez les données du relevé topo et ses croquis dans votre format préféré et transférez les fichiers exportés sur votre PC, pour terminer le dessin de la topographie de la cavité avec votre programme de dessin habituel.
Par sécurité, créez une archive zip de TopoDroid et transférez-la sur votre PC. L'archive zip est une sauvegarde complète de votre relevé, et elle contient la commande permettant de restaurer les données du relevé et les autres informations de la base de données. Vous pouvez l'utiliser pour transférer le relevé topographique de votre appareil TopoDroid vers un autre périphérique Android (tablette, smartphone...).

Les données du relevé, ainsi que l'archive zip, sont exportées à partir de la fenêtre d'information du relevé.
Les croquis du relevé sont exportés à partir de la fenêtre de croquis (un seul croquis) et de la fenêtre de vue d'ensemble (tous les croquis dans un seul fichier).

Les archives zip de TopoDroid sont stockées dans le dossier zip.

Les fichiers de données et de croquis exportés sont stockés dans des dossiers nommés de manière appropriée sous le dossier "survey".

Attention : L'importation du Zip échoue si le Zip a une version de base de données trop ancienne ou plus récente que celle de TopoDroid. La compatibilité à l'importation avec un autre format de fichier de topographie peut ne pas être complète.

TopoDroid importe des données du relevé topo à partir d'archives zip TopoDroid ou de fichiers externes.
Pour importer des archives zip à partir du dossier interne, appuyez sur le bouton "Importer" de la fenêtre principale.
Pour importer des fichiers externes (ou des archives zip externes), appuyez longuement sur le bouton "Importer". Les fichiers externes sont sélectionnés par un module du système.
Le nom du fichier ne doit pas contenir de signe le ponctuation deux points. ':'.
Les formats d'importation suivants sont pris en charge :

  • Compass
  • Survex
  • Therion. (seulement les données)
  • VisualTopo
  • CaveSniper
  • PocketTopo (y compris les croquis)
  • Archives zip de TopoDroid

L'importation d'un fichier échoue si un relevé portant le même nom se trouve déjà dans TopoDroid.
L'importation d'une archive zip échoue si le zip possède une version de la base de données trop ancienne ou plus récente que celle de TopoDroid.

Lors de l'importation à partir de formats de fichiers qui ont des valeurs LRUD, celles-ci sont converties en valeurs x (affichées en vert).

Importation de Therion
Seuls les fichiers de relevés Therion avec des données en mode de données "normal" sont importables.
La commande "declination" est prise en compte : les azimuts lus sont corrigés avec la valeur de la déclinaison magnétique.
TopoDroid n'importe pas les grilles Therion.


Importation de PocketTopo
TopoDroid importe à la fois les données du relevé et les croquis. Cet import fait correspondre les couleurs PocketTopo aux outils de dessin de TopoDroid (points et lignes). Le mappage peut être modifié par un paramètre.


Importation depuis Compass
TopoDroid convertit les relevés de données importés depuis Compass du mode "normal" au mode "plongée" [T], si le paramètre correspondant est activé [G].

NB : L'importation peut échouer (et faire planter l'application) en raison d'une défaillance de la base de données que TopoDroid ne peut pas intercepter. Si cela se produit, supprimez le relevé partiellement importé et essayez de l'importer à nouveau.

FORMATS D'EXPORT PAR DÉFAUT

TopoDroid possède deux paramètres pour sélectionner un format d'exportation par défaut pour les données de topographie et un seul pour les croquis. Par défaut, ils sont tous les deux réglés sur "none", c'est-à-dire sans exportation.
Lorsque le format d'exportation des données de levé topographique est défini, TopoDroid exporte les données du levé topo dans ce format avant de créer une archive zip, et ce afin que l'exportation soit incluse dans l'archive.

Lorsque le format d'exportation d'esquisse par défaut est défini, les croquis sont exportées dans ce format à chaque fois qu'elles sont fermées (en plus d'être enregistrées au format binaire TopoDroid).

EXPORTATION DES DONNÉES DU RELEVÉ

Remarque : Les points fixes sont exportés avec leurs coordonnées latitude-longitude [en degrés décimaux] et l'altitude au-dessus du niveau moyen de la mer [m].

TopoDroid exporte les données des relevés topographiques aux formats des programmes de topographie spéléo suivants :

  • Compass
  • cSurvey
  • Survex
  • Therion
  • TopoRobot
  • VisualTopo
  • Walls
  • WinKarst
  • GHTopo
  • Grottolf
  • PocketTopo
  • Polygon
  • Topo
En outre, il exporte les données des relevés topographiques informatiques aux formats suivants :
  • CSV
  • DXF
  • KML
  • GeoJSON
  • trackfile (OziExplorer)
  • shapefile

Si le relevé n'est pas géoréférencé, les données du relevé ne peuvent pas être exportées dans les quatre derniers formats d'exportation.

Compass (.dat)
Fichier de données Compass sont pris en charge à la fois à l'import et à l'export.
Les fichiers importants de Compass doivent avoir l'extension ".dat" ou ".DAT".
Les unités de données sont 'pieds' et 'degrés'.

Paramètres

  • Préfixe de la station : s'il est mentionné, il est ajouté au début du nom de la station.
  • Visées latérales d'habillage (rayonnement)
  • Échange G/D

cSurvey (.csx)
Les fichiers d'exportation cSurvey nécessitent cSurvey v. 1.20 ou ultérieur.
L'exportation cSurvey à partir de la fenêtre d'information du relevé ne comprend que les données du relevé, et ne comporte pas de croquis.
Pour réaliser une exportation cSurvey avec un croquis (plan et coupe), vous devez exporter à partir de la fenêtre de dessin.

L'exportation cSurvey inclut les visées de contrôle d'étalonnage et les fichiers média (audio et photo).

Paramètres :

  • Préfixe de la station : préfixe ou non le nom de la station avec le nom du relevé.

Survex (.svx)
Les données du relevé peuvent être exportées sous forme de fichiers Survex.
Les unités de données suivent les options de l'utilisateur.

Paramètres :

  • la fin de ligne peut être soit du type Unix, soit du type Windows
  • station nommée TO pour les visées d'habillage (le nom TO devient le nom de la station FROM avec une lettre en annexe)
  • ajouter les visées LRUD à l'exportation

Therion (.th)
Les données du relevé topo sont exportées au format Therion, en utilisant le formulaire "normal".
Les unités de données sont définies par les options de l'utilisateur.
Les noms des équipes sont répertoriés mais la commande est commentée.
Les commentaires et les marques de la station (fixes, couleurs) précèdent les données de la ligne de cheminement.
Les visées latérales sont exportées avec la station "-" (tiret), les autres avec la station "." (point).

Les coordonnées des points fixes (latitude longitude et altitude a.s.l.) sont exportées commentées. Si le point a des coordonnées CRS personnalisées, celles-ci sont également exportées, précédées du nom CRS.

De plus, les "inputs" des fichiers th2 et les définitions pour la topographie de chaque calque (voir doc. Therion) sont ajoutés à la fin, mais sont commentés.

Paramètres

  • Cartes Therion : si cette option est activée, les commandes "map" sont placées avant la ligne de cheminement du relevé topographique.
  • LRUD : ajouter LRUD pour exporter.

Les fichiers import Therion doivent avoir l'extension ".th".
L'analyse TopoDroid de la syntaxe des données Therion est très limitée. Généralement, seules les données sous forme "normale" sont comprises. Les commandes de données Therion "units", "flag", "extend", "mark" et "station" sont supportées.
Les points fixes ("fix") sont analysés mais ne sont pas pris en compte car "cs" n'est pas supporté.

TopoRobot (.trb)
Les données du relevé peuvent être exportées sous forme de fichiers TopoRobot. La stratégie de dénomination des stations de TopoRobot doit avoir été sélectionnée. Si les noms de station suivent  la convention TopoRobot "numéros de série", ils sont alors utilisés.

VisualTopo (.tro)
Les fichier de données VisualTopo sont pris en charge à la fois à l'import et à l'export.
Les fichiers VisualTopo importés doivent avoir l'extension ".tro" ou ".TRO".
Les unités utilisées sont 'm' et 'degrés'.

Paramètres :

  • Format tro. Par défaut, les données sont enregistrées au format "tro".
  • Visées latérales d'habillage
  • LRUD liées à la station DEPUIS (FROM)
  • Suffixe de la station : s'il est mentionné, il est ajouté aux noms des stations.

Walls (.srv)
Les données du relevé peuvent être exportées sous forme de fichiers Walls (partiellement testés).
Les unités sont données selon les options de l'utilisateur.

WinKarst (.sur)
Les données du relevé peuvent être exportées sous forme de fichiers WinKarst (non testé).

Paramètres :

Préfixe de la station : s'il est donné, il est ajouté au début du nom de la station.

GHTopo (.gtx)
TopoDroid peut exporter des données de relevé topographique au format GHTopo (partiellement testé).

Grottolf (.grt)
TopoDroid peut exporter des données topographiques au format Grottolf (non testé). Si la première station a des coordonnées géographiques, celles-ci sont exportées, sinon les coordonnées locales sont utilisées. La coupe sur les stations est calculée en projetant visées d'habillage sur un plan vertical perpendiculaire au cheminement.

PocketTopo (.top)
TopoDroid prend en charge PocketTopo aussi bien à l'import qu'à l'export.

Les exportations ne contiennent que les données du relevé topo.

Si la stratégie d'attribution des noms de station, qui est utilisée dans l'export, est réglée sur "TopoRobot" et que les noms de stations suivent la convention TopoRobot "series.number", alors les noms sont convertis en nombres. Les caractères alphabétiques remplacent chaque caractère par deux chiffres. Plus précisément "a" devient"00", "b" devient"01" et ainsi de suite, "A" devient"50", "B" devient"51", et ainsi de suite. Chaque autre caractère (c'est-à-dire non alphanumérique) passe à "99".

Les fichiers importés depuis PocketTopo doivent avoir l'extension ".top" ou ".TOP".
Les dessins du "plan" et de la "coupe" sont importés. PocketTopo n'utilise que peu de couleurs pour différencier les types de ligne. Par conséquent la palette de couleurs de Therion est utilisée. Les types de ligne et de point peuvent être réglés via un paramètre.

Il peut arriver que le croquis importé ne soit pas aligné avec la topographie issue des données du relevé. Dans ce cas, vous pourrez déplacer l'esquisse pour l'adapter à l'axe de cheminement (option Références fixes du menu déroulant).

Polygon (.cave)
TopoDroid peut exporter des données de relevé topographique au format Polygon. Les points fixes sont exportés en tant que lat-long.

Topo (.cav)
Les données du relevé peuvent être exportées sous forme de fichiers Topo.

Paramètres :

  • Fin de ligne peuvent être soit du type Unix soit du type Windows

Fichier CSV (.csv)
Les données du relevé peuvent être exportées sous la forme d'une liste d'enregistrements avec les champs séparés par des virgules :

  • Station"Depuis"
  • Station "Vers"
  • Distance, azimut, pente
  • Icône si présente

DXF (.dxf)
Les données du relevé sont exportées sous forme de couches 3D DXF

  • Stations
  • Cheminement
  • Visées d'habillage

Les données sont corrigées avec la déclinaison magnétique.

KML (.kml)
Les données du relevé peuvent être exportées en tant que fichier KML (GoogleEarth) si vous avez géoréférencé au moins une station. Si le relevé topographique contient des tronçons disjoints, l'export KML inclut tous les tronçons ayant une station géoréférencée. Les données sont corrigées avec la déclinaison magnétique.

Paramètres :

  • Stations
  • Visées d'habillage

GeoJSON (.json)
Les données du relevé topographique peuvent être exportées en tant que fichier GeoJSON (RFC 7946) si vous avez géoréférencé au moins une station. Les prises de vue sont exportées sous forme de LineString, les stations sous forme de Point. Si le levé contient des morceaux disjoints, l'exportation GeoJSON inclut tous les morceaux avec une station géoréférencée. Les données sont corrigées avec la déclinaison magnétique.

Paramètres :

  • Stations
  • Visées d'habillage


PLT (.plt)
Les données du relevé peuvent être exportées en tant que fichier de trace (OziExplorer) si vous avez géoréférencé une station. Si le relevé topographique contient des tronçons disjoints, le fichier de trace inclut tous les tronçons ayant une station géoréférencée. Les données sont corrigées avec la déclinaison magnétique.

Shapefile (.shp .shx .dbf)
Les données du relevé topographique peuvent être exportées sous forme de shapefile. Il y a un fichier pour les stations et un autre pour les visées (à la fois de cheminement et d'habillage). Si le relevé contient des pièces disjointes, un shapefile est exporté pour chaque pièce avec une station géoréférencée. Les données sont corrigées avec la déclinaison magnétique. Chaque shapefile est composé de trois fichiers : le fichier de données (shp), le fichier d'index (shx) et le fichier d'attributs (dbf). Les fichiers sont compressés en zip et le .zip est enregistré dans le dossier "shp" avec l'extension ".shz".

Si vous avez géoréférencé au moins une station et coché la case appropriée dans le dialogue d'exportation, les données sont exportées dans le système de référence WGS84, et corrigées avec la déclinaison magnétique.
Sinon, les données sont exportées en coordonnées locales et la correction de la déclinaison magnétique n'est pas appliquée.

Le fichier .dbf  "stations" contient uniquement le champ "nom".
Le fichier .dbf  "visées" contient les champs suivants :

  • type : soit "cheminement", soit "habillage".
  • DEPUIS la station
  • VERS la station ("-" pour les visées d'habillage)
  • indicateur de la visée sous forme hex : 1-visée de surface, 2-visée dupliquée, 4-visée commentée
  • commentaires des visées

Paramètres :

  • Stations : si vous voulez exporter les stations
  • Visées d'habillage : inclure ou non les visées latérales d'habillage (rayonnement)

 

EXPORTATION de CROQUIS

TopoDroid exporte les croquis des relevés topographiques aux formats suivants :

  • cSurvey
  • Therion
  • Tunnel
  • xtherion
  • Cave3D
  • DXF
  • PDF file
  • shapefile
  • SVG

cSurvey (.csx)
Ces exportations contiennent à la fois les croquis de plan et de coupe, ainsi que les données de relevés topographiques. Lorsque des points "section" sont définis, les dessins de section transverses sont ajoutés à ces points.
L'attribut cSurvey "cave" est défini avec le nom du relevé topographique et l'attribut "branch" avec le nom du croquis. Les branches, les séparations de ligne de cheminement et les éléments d'esquisse partagent ces attributs.

Si votre relevé topographique ne contient qu'une seule esquisse, il s'agit d'une exportation complète de votre travail (sauf pour les sections transverses). Si le relevé topographique contient plus d'un croquis, vous devrez exporter chaque esquisse individuellement et fusionner les exportations avec cSurvey.

Les éléments des esquisses sont répartis sur des couches cSurvey appropriées.

Therion (.th2)

les croquis sont exportées au format Therion sous la forme d'une seule esquisse par fichier, si elles sont exportées depuis la fenêtre d'esquisse, ou sous la forme d'un fichier contenant toutes les croquis avec la même vue, en plan ou coupe développée, si elles sont exportées depuis la fenêtre d'aperçu. Dans ce dernier cas, il y a un calque Therion (scrap) pour chaque esquisse, et les scrap de section sont inclus si les points de section automatiques sont activés.

Les noms de scrap Therion sont composés du nom du relevé, du nom de l'esquisse et d'un suffixe, soit 'p' pour plan, soit 's' (du mot "section") pour la coupe.
Pour les croquis multi-scrap, les noms des scraps Therion, au-delà du premier, sont également accompagnés d'un numéro commençant par 1.
Les noms de scrap Therion des sections transverses sont composés du nom du relevé, d'un code de deux lettres, 'xx', et d'un numéro.

Lorsqu'un croquis avec des points de section transverse est exportée via la boîte de dialogue d'exportation, les sections transverses des points de section sont également exportées.

Si l'esquisse contient des symboles qui ne sont pas inclus dans le jeu de symboles Therion, vous devez ajouter le préfixe "u :" aux noms des symboles non supportés, avant d'utiliser le fichier th2.

Paramètres :

  • points de station automatiques. [par défaut : yes].
  • ajout de visées d'habillage au scrap (sous forme de lignes de type "splay") [par défaut : no].
  • ajouter une commande Xtherion pour lire l'image XVI du croquis [par défaut : non].
  • exporter l'échelle. Par défaut : 1:100
  • espacement des points : distance minimale entre les points sur les polylignes. Les points intermédiaires ne sont pas exportés. [par défaut : 20 cm]

Les images XVI ne sont pas automatiquement exportées avec le fichier Therion "th2", même si une commande Xtherion pour l'inclure est ajoutée au fichier Therion, car elle ne contiendrait pas plus d'informations que le fichier Therion lui-même. La commande peut être utilisée pour charger dans Xtherion une image XVI créée par un autre programme.

Remarque. Avec la version 3.1, TopoDroid enregistre les croquis "nativement" dans son format binaire. Les versions précédentes utilisaient le format Therion. La conversion de Therion en binaire est "automatique" lorsque vous ouvrez un croquis : si TopoDroid ne trouve pas le fichier binaire, il recherche le fichier Therion.

Tunnel (.xml)
Sketch peut être exporté au format Tunnel XML. Les fichiers sont enregistrés dans le sous-répertoire "tnl".
Les lignes suivantes sont mappées aux lignes du Tunnel : mur, présumé, fosse, cheminée, et pente. Toutes les autres lignes sont mappées à la ligne "details" de Tunnel.
Les limites de la zone sont mappées à la ligne "filled" de Tunnel.
Les points suivants sont associés aux points du tunnel : courant d'air, archéo, blocs, argile, colonne, rideau, pierre d'écoulement, gradient, guano, boue, cailloux, popcorn, sable, soda-paille, stalactite, stalagmite, eau, écoulement d'eau. Tous les autres points sont mappés au point de Tunnel "bedrock".


Xtherion (.xvi)
L'exportation "xvi" peut être insérée comme image de fond dans Xtherion.
Les lignes de type "paroi" sont rouges, "pente" orange, "section" grise, toutes les autres sont brunes. Les bordures des zones sont noires. Les icônes de points sont rendues de manière simplifiée. Le rendu des étiquettes ne supporte que les caractères latins, les chiffres et quelques caractères ('+', '-', '/', '_', '>', '<', '?'). Les autres caractères sont remplacés par un "losange".
La maille de la grille est au pas de 1 m.

Les paramètres XVI font partie des paramètres Therion :

  • Dessin des points de section : si des points de section sont automatiquement ajoutés aux lignes de section, le dessin de la section transverse est inséré dans le fichier xvi, au point "section".
  • lignes de grille ajoutées à l'exportation (taille de la cellule correspondant aux unités de grille courantes). [par défaut, aucune grille]
  • échelle d'exportation. [par défaut 1:100]

Cave3D (.c3d)
L'esquisse exportée au format "c3d" peut être transposée en modèle 3D grâce au visualisateur 3D.

DXF (.dxf)
Les croquis (esquisses) peuvent être exporté(e)s sous forme de fichiers DXF 2D avec des couches pour chaque point, ligne et type de zone en plus des couches de données 3D DXF. Les éléments du croquis 2D DXF sont situés dans le plan z =0.
les croquis peuvent être exportées en tant que DXF 9 AC1009, 12 (R13) AC1012, et 14 (R14) AC1014.
Pour le DXF-9 les zones sont exportées avec seulement la bordure et ne sont pas colorées à l'intérieur. Il est possible d'inclure dans le fichier DXF, les sections tranxverses (pour lesquelles l'esquisse contient un point "section") et l'axe X-Y.

Paramètres :

  • Version DXF : 9 (dxf-9), 12 (dxf-13), ou 14 (dxf-14). [par défaut "9"].
  • Point de section : si une ligne de section transverse possède le point de section relatif, le dessin de la section est inséré dans l'exportation au point "section".
  • Référence XY : insertion ou non de la référence des axes XY
  • Stations : insertion ou non des points de station
  • Courbes interpolées : permet de dessiner les cubiques comme des polylignes interpolées.
  • Espacement des points : distance maximale entre les points d'interpolation sur les lignes lissées. [par défaut 20 cm]

PDF (.pdf)
les croquis peuvent être exportées sous forme de fichiers PDF. Les fichiers PDF de croquis (oun esquisses) sont conçus comme un moyen rapide de partager la topographie avec des non spéléologues dans un format que tout le monde peut ouvrir.

Le fichier PDF a une seule page sur fond blanc et il comprend uniquement les éléments de référence (stations, visées d'habillage, grille, etc.) qui sont affichés à l'écran.
Le taille du texte des étiquettes, des indications des barres d'échelle et des stations, est réduite d'un quart. La couleur de certains éléments (par exemple des étiquettes) est inversée, pour plus de visibilité.

Fichier Shapefile (.shp)
les croquis peuvent être exportées en tant que shapefile, avec des stations, des visées et des lignes. Chaque élément possède trois fichiers : le fichier de données de forme, le fichier d'index et le fichier de base de données. Les fichiers composant l'exportation shapefile sont compressés par zip et le fichier .zip est enregistré dans le dossier "shp" avec l'extension ".shz".

Si vous avez géoréférencé au moins une station et coché la case appropriée dans le dialogue d'exportation, les données sont exportées dans le système de référence WGS84, et corrigées avec la déclinaison magnétique.
Sinon, les données sont exportées en coordonnées locales et la correction de la déclinaison magnétique n'est pas appliquée.

Le fichier .dbf  "points" contient uniquement les champs suivants :

  • nom du point
  • orientation
  • niveau
  • scrap
  • texte optionnel (tronqué à 128 caractères)

Le fichier .dbf  "extra" contient les champs nom du point :

  • orientation
  • niveau
  • scrap
  • texte facultatif (tronqué à 128 caractères)
  • fichier

Les fichiers .dbf  "line" et "area" contiennent :

  • le type qui permet de distinguer la ligne de la zone
  • nom de la ligne ou de la zone
  • niveau
  • scrap  

Le fichier .dbf  "station" contient :

  • le nom de la station.

Le fichier .dbf  "visées" contient les champs suivants :

  • type : soit "cheminement", soit "habillage".
  • la station DEPUIS
  • la station VERS ("-" pour les visées d'habillage)
  • un indicateur de la visée sous forme hex : 1-visée de surface, 2-visée dupliquée, 4-visée commentée

SVG (.svg)
Les croquis peuvent être exportés sous forme de fichiers SVG.

Les éléments sont organisés en couches : grilles, stations, visées de cheminement, visées d'habillage, points, lignes et zones.
Ces couches (layers) se trouvent sous une couche de calque global.
Chaque "scrap" a ses propres couches points, lignes et zones.
L'échelle d'exportation est de 1:100.

Si les classes d'affichage sont activées, les visées d'habillage sont colorées en fonction de leur classe : les visées d'habillage normales sont gris, les horizontales sont vert marin clair, les verticales sont bleu acier et les sections transverses sont bleu ciel. Les classes d'habillage ne sont pas prises en charge dans l'exportation type "aller-retour".

Les effets de ligne, tels que les coches sur les parois de puits, ne sont pas exportés, mais la direction peut être montrée en activant le paramètre "Line Direction".

Paramètres :

  • Programme SVG : Inkscape ou Illustrator. Cela affecte un facteur d'échelle d'exportation.
  • Point de section du dessin : si des points de section sont automatiquement ajoutés aux lignes de section, le dessin de section est inséré dans l'export, au point "section".
  • Points de station automatiques : si cette option est activée, toutes les stations sont incluses dans l'exportation, sinon seules les stations choisies par l'utilisateur sont incluses.
  • Espacement des points : distance maximale entre les points d'interpolation sur les lignes lisses
  • Round-trip : les croquis sont exportées prêtes pour le round-trip de Walls/Compass [non]
  • Grille : quadrillage ajouté à l'export (taille des carreaux calquée sur les unités de la grille en cours). [non]
  • Orientation des ignes : coche [non]
  • Épaisseur des lignes : étiquettes, icônes, lignes (et limites de zone), lignes du quadrillage de la grille, visées, trait d'orientation
  • taille des stations : taille du texte des noms de stations. [par défaut : 20]

 

EXPORTATION D'ÉTALONNAGE

    Les données d'étalonnage peuvent être exportées et importées au format CSV (fichiers.csv).

PARAMÈTRES d'IMPORT/EXPORT

  • PocketTopo import color-map : palette des couleurs PocketTopo pour esquisser les types d'éléments.
  • Estimation LR extend : à l'importation des fichiers Compass / VisualTopo [oui]
  • Format d'exportation de données par défaut : définit le format par défaut auquel les données sont exportées. Ce fichier est généré automatiquement et inséré dans l'export ZIP. [défaut non]
  • Format d'exportation du croquis par défaut : définit le format par défaut auquel les croquis sont exportées. Ces fichiers sont automatiquement générés chaque fois qu'un croquis est fermée. Cela impose une charge supplémentaire pour le système, vous pouvez donc opter pour l'exportation explicite d'esquisses après la session de levé topographique. [défaut non]
  • Exportation automatique des esquisses. Si cette option est activée, les croquis sont exportés dans le format choisi lorsqu'ils sont fermés. Les fichiers exportés sont stockés dans le sous-dossier "out" du dossier de l'enquête. L'exportation automatique des croquis impose une charge de travail supplémentaire à l'application. [défaut non]
  • Commandes de carte Therion avant ou après le bloc de ligne centrale [après]
  • Stations automatiques Therion : ajoutez automatiquement des points de station au(x) fichier(s) d'esquisse Therion [oui]. Si vous voulez utiliser les fichiers d'esquisse dans Therion, ils doivent avoir des points de station pour définir leur échelle. Vous pouvez choisir manuellement le point à ajouter (en éditant le dessin d'esquisse), ou laisser TopoDroid ajouter les stations. Le programme ajoutera toutes les stations qui sont "à l'intérieur" du dessin (à l'intérieur du dessin de l'enveloppe convexe).
  • Lignes u:splays Therion dans les calques de Therion [oui]
  • cSurvey / Compass préfixe du relevé topo sur le site du relevé topo aux stations [non]
  • Compass / TopoRobot / WinKarst / Polygon / VisualTopo
    • Angle minimum de la visée latérale pour le calcul LRUD [0.0] Ceci est également utilisé dans Survex / Therion si les LRUD sont requis.
    • minimum clino des visées utilisées pour UP / DOWN dans LRUD [0]
    • pente maximale des visées utilisées pour LEFT /RIGHT dans LRUD [90]
  • Compass swap LR : échange L et R sur l'exportation de Compass.
  • Terminaison de ligne Survex / Topo : Unix ou Windows [Unix]
  • Exportation Survex avec les stations named splay TO [non]
  • Exportation Survex / Therion avec des lignes LRUD [non]
  • Exportation SVG avec grille [non]
  • L'orientation de la ligne d'export SVG est activée [non]
  • Largeurs de trait SVG (étiquettes, icônes, lignes, grilles, plans, coches d'orientation)
  • Exportation KML/GeoJSON avec les stations [oui]
  • Export KML/GeoJSON avec des les visées latérales d'habillage [non]
  • Échelle d'exportation d'esquisse DXF [1.0]
  • Exportation DXF avec les points de terminaison des visées latérales d'habillage [non]
  • Version DXF AutoCAD pour l'exportation DXF : 6 (AutoCAD R9) ou 12 (AutoCAD R13). Dans la version 6, les courbes Bezier d'exportation DXF sont remplacées par des polylignes et les zones n'ont pas de hachures. La version 12 de l'exportation DXF utilise des splines et des hachures, mais elle est toujours boguée.
