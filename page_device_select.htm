APPAREILS DE MESURE (ou DISPOSITIFS) PÉRIPHÉRIQUES

TopoDroid supporte les appareils suivants :

  • DistoX (première version basée sur Leica DistoA3)
  • DistoX2 (deuxième version basée sur Leica DistoX310)
  • Sap5 (Shetland Attack Pony), seulement les modèles avec Bluetooth, fabriqué en GB
  • Bric4 (Bluetooth Ruggerized Integrated Cartographer), fabriqué aux USA

Tous les appareils doivent être appairés avec Android.

Normalement, TopoDroid ne fonctionne qu'avec un seul appareil à la fois. Les données peuvent être téléchargées en continu (mode continu) ou en lot (mode à la demande). La différence entre ces deux modes est que le mode continu reste actif lorsque le téléchargement des données est terminé ou que la connexion avec l'appareil est perdue, tandis que le mode à la demande est fermé.

Sap5 et Bric4 ne téléchargent les données qu'en mode continu.

Avec TopoDroid et le DistoX, il est possible de travailler avec plusieurs appareils en même temps, bien que les données ne soient téléchargées que depuis un seul appareil à la fois.
Le mode de téléchargement des données doit être réglé sur multi. Lorsque c'est le cas, un appui long sur le bouton de téléchargement fait apparaître une boîte de dialogue permettant de choisir le DistoX à partir duquel télécharger les données.

 

Shetland Attack Pony
TopoDroid supporte le modèle du Shetland Attack Pony, version 5 (Sap5), avec le Bluetooth.
L'appareil doit avoir été préalablement "appairé" avec Android, en utilisant le menu Paramètres de l'application. Le code PIN est "000000" (six zéros).
Ensuite, le Sap5 est répertorié parmi les appareils connus dans la fenêtre des appareils périphériques. Il apparaît comme "SAP XX MAC-address", où "XX" est le code à deux lettres du Sap5, par exemple "HG".
Pour sélectionner le Sap5 comme dispositif actuel, tapez sur son entrée dans la liste.

Lorsqu'il est connecté, le Sap5 transfère la lecture des valeurs de visées à l'application, que les données de ces visées soient stockées ou non sur le dispositif. Le protocole de transmission de données de Sap5 ne comprend pas d'accusé de réception des données. Par conséquent, si le Sap5 n'est pas connecté, les valeurs des visées ne sont pas transférées à l'application.

ATTENTION : lors de sa reconnexion, le Sap5 ne transfère pas les données des visées stockées lorsqu'il n'était pas encore connecté.

Le Sap5 conserve dans sa mémoire interne les données qui ont été "stockées". Celles-ci peuvent être récupérées sous forme de fichier Survex, en utilisant le programme Sap PonyTrainer.

L'étalonnage du Sap5 est effectué par l'appareil lui-même et ne nécessite pas de programme externe.

 

Bric4
TopoDroid supporte la version 4 du Bric.
L'appareil doit avoir été préalablement "appairé" avec Android, en utilisant le menu Paramètres de l'application.
Ensuite, le Bric4 est répertorié parmi les appareils connus dans la fenêtre des appareils. Elle apparaît comme "Bric4 code MAC-address", où le code est le code à quatre chiffres du Bric4, par exemple "0039".
Pour sélectionner le Bric4 comme appareil courant, tapez sur son entrée dans la liste.

Une fois connecté à un client, le Bric4 se souvient de la connexion, même s'il est éteint, il reste donc "toujours connecté".
Lorsque TopoDroid est connecté, les relevés des visées sont immédiatement transférés vers Android. Si TopoDroid n'est pas connecté, les données sont conservées en mémoire et transférées lors de la prochaine connexion de l'application.

Le protocole de transmission des données Bric4 inclut un accusé de réception pour Android, mais il n'inclut pas d'accusé de réception des données par l'application. Le protocole prévoit un délai de quelques secondes entre la connexion et le premier transfert de données et TopoDroid prévoit un délai d'une seconde après la dernière donnée avant de fermer la connexion. De cette manière aucune photo n'est perdue, bien que cela ne soit pas garanti et que des photos soient perdues si l'application n'est pas notifiée par Android avant l'arrivée de la photo suivante.

Le Bric4 stocke les données dans la mémoire interne dans des fichiers CSV. S'il est connecté à un PC via USB, l'appareil apparaît comme une mémoire externe et les fichiers CSV peuvent être copiés sur le PC.

L'étalonnage du Bric4 est effectué par l'appareil et ne nécessite pas de programme externe.

Le Bric4 peut être contrôlé à partir de l'application TopoDroid. Il est notamment possible d'allumer/éteindre le laser, et de prendre des photos.

Le Bric4 dispose également d'un "mode scan" dans lequel les relevés sont effectués en continu, environ deux par seconde.

TopoDroid peut suivre trois protocoles d'acquisition de données différents (selon un réglage personnalisé "custom set"). Le premier ne reçoit que les données primaires (distance, azimut et pente) qui ne comprennent pas le numéro d'identification des données attribué par le Bric4. Les deux autres reçoivent les données complètes. La différence est que TopoDroid utilise le numéro d'identification fourni par le Bric4 ou attribue ses propres numéros aux visées.

Lorsque les données complètes sont reçues, TopoDroid enregistre les codes d'erreur et les valeurs du Bric4 dans le commentaire de la visée.
